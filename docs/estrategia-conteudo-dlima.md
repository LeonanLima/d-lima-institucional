## PARTE 5 — MODELO DE 3 LINHAS DE SERVIÇO

### Veredito do Conselho Estratégico

**A questão central:** Como estruturar, comunicar e vender o modelo de 3 linhas da D'LIMA com Trava de Segurança obrigatória nos projetos >200m².

**Onde o conselho convergiu:**
- A Trava de Segurança é o **ativo mais valioso** do modelo, não uma restrição burocrática
- Metragem como critério único tem limitações reais — entrevista de escopo complementa como inteligência interna
- O maior gap atual é credibilidade/histórico — o modelo precisa ser construído com honestidade sobre capacidade operacional real
- A estrutura de 3 linhas cria um **funil de ascensão com alto potencial de LTV composto**

**A tensão central:** "Prove um nível antes de lançar três" vs. "estruture e lance — o framework é o diferencial". Resolução: **lance a estrutura completa, mas operacionalize progressivamente** — documente os processos de cada linha antes de qualquer material comercial.

**Por que funciona:**
- Critérios claros são defesa legal, comercial e de reputação ao mesmo tempo
- A Trava não expulsa bons clientes — expulsa clientes que querem negociar qualidade em obra complexa (os mais custosos operacionalmente)
- O modelo Essencial→Conforto→Exclusive cria valor composto: quem entra no Essencial com boa experiência volta e indica
- Metragem como triagem é correto para o mercado; complexidade como critério interno é a inteligência real

**Riscos a vigiar:**
- A fronteira 199→201m² vai gerar pressão — prepare zona de conversa proativa para clientes na faixa 175–199m²
- Não lance as 3 linhas com materiais idênticos — cada uma precisa de processo documentado real
- Cliente Exclusive espera que as regras "dobrem para ele" — o script de venda precisa transformar a rigidez em benefício antes da objeção chegar

**Próximo passo concreto (Executor):** Uma folha A4 por linha respondendo: quem atende, quantas reuniões estão incluídas, qual o prazo de resposta garantido, quais são os entregáveis. Isso vira marketing e contrato ao mesmo tempo.

> **Insight:** O modelo de 3 linhas aplica o *Decoy Effect* de pricing psychology: a Linha Conforto torna a Exclusive razoável por contraste, enquanto a Essencial ancora o piso. A Trava de Segurança é uma *authority frame* (Klaff) — posiciona a D'LIMA como o *prize*, não como *supplicant*. Isso é psicologicamente poderoso porque inverte o padrão do mercado de construção, onde a construtora sempre cede.

### Simulador de Categoria (lógica Python)

```python
def classificar_projeto(metragem_m2: float, desejo_cliente: str = None) -> dict:
    LINHAS = ["essencial", "conforto", "exclusive"]
    TAXAS_BASE = {
        "essencial": {"base": 0, "por_m2": 0},   # preencher com valores reais
        "conforto":  {"base": 0, "por_m2": 0},
        "exclusive": {"base": 0, "por_m2": 0},
    }

    # TRAVA DE SEGURANÇA (inviolável)
    if metragem_m2 > 200:
        return {
            "categoria": "Linha Exclusive",
            "trava_ativa": True,
            "upsell_aplicado": False,
            "justificativa": (
                "Projetos acima de 200m² requerem protocolo "
                "Concierge de Engenharia obrigatoriamente. "
                "Trava de Segurança ativa — risco técnico elevado."
            ),
            "taxa_estimada": calcular_taxa("exclusive", metragem_m2, TAXAS_BASE),
        }

    # CLASSIFICAÇÃO BASE POR METRAGEM
    if metragem_m2 <= 70:
        categoria_base = "essencial"
    else:  # 71–199
        categoria_base = "conforto"

    # UPSELL DE CONVENIÊNCIA
    upsell_map = {
        "essencial": ["conforto", "exclusive"],
        "conforto":  ["exclusive"],
        "exclusive": [],
    }

    categoria_final = categoria_base
    upsell_aplicado = False

    if desejo_cliente and desejo_cliente.lower() in LINHAS:
        desejo = desejo_cliente.lower()
        if desejo in upsell_map[categoria_base]:
            categoria_final = desejo
            upsell_aplicado = True
        elif LINHAS.index(desejo) < LINHAS.index(categoria_base):
            return {
                "categoria": categoria_base.capitalize(),
                "alerta": (
                    f"Seu projeto ({metragem_m2}m²) exige no mínimo "
                    f"a Linha {categoria_base.capitalize()}. "
                    "Downgrade não disponível."
                ),
                "trava_ativa": False,
                "upsell_aplicado": False,
            }

    nome_linha = {
        "essencial": "Linha Essencial",
        "conforto":  "Linha Conforto",
        "exclusive": "Linha Exclusive",
    }

    return {
        "categoria": nome_linha[categoria_final],
        "categoria_base": nome_linha[categoria_base],
        "trava_ativa": False,
        "upsell_aplicado": upsell_aplicado,
        "taxa_estimada": calcular_taxa(categoria_final, metragem_m2, TAXAS_BASE),
    }


def calcular_taxa(categoria: str, metragem: float, taxas: dict) -> float:
    t = taxas[categoria]
    return t["base"] + (metragem * t["por_m2"])


# Nota VBA: mesma lógica com If metragem > 200 Then categoria = "Exclusive" : GoTo Saida
```

### 5 Argumentos de Justificativa de Valor

**Arg. 1 — Risco técnico não-linear (técnico)**

Projetos acima de 200m² não são "obras maiores" — são obras com geometria de risco diferente. Uma decisão equivocada em estrutura, instalações ou sequenciamento de subempreiteiros em 250m² tem impacto em cascata que pode comprometer prazos, orçamento e segurança estrutural.

*Argumento de venda:* "A diferença entre o custo do Exclusive e o custo de um retrabalho estrutural em 200m²+ não é percentual. É uma ordem de grandeza."

---

**Arg. 2 — Responsabilidade técnica registrada (técnico-legal)**

O engenheiro responsável assina a ART de projeto e execução. Gerir uma obra de 220m² com protocolo de projeto de 80m² não é questão de escolha operacional — é risco legal materializado em responsabilidade civil e profissional. A Trava protege o cliente tanto quanto protege o profissional.

*Argumento de venda:* "Quando eu assino a responsabilidade técnica, estou formalmente comprometido com cada decisão. O Exclusive é o protocolo que torna esse comprometimento sustentável."

---

**Arg. 3 — Ancoragem no custo total do projeto (financeiro)**

Uma obra de 200m²+ raramente representa menos de R$ 800k–R$ 1,5M em custo de construção. A diferença entre Conforto e Exclusive representa percentual pequeno do custo total — e garante gestão especializada de 100% desse orçamento.

*Argumento de venda:* "Você está investindo R$ 1 milhão em construção. O custo do Exclusive representa X% desse investimento. Qual é o custo de não ter esse protocolo?"

---

**Arg. 4 — Psicologia da perda, não do ganho (psicológico)**

A pergunta certa não é "quanto custa a Exclusive?" — é "quanto custa não ter o protocolo Exclusive em um projeto desse tamanho?" Atraso de cronograma tem custo de hospedagem/aluguel. Retrabalho tem custo material. Conflito de fornecedores tem custo de estresse.

*Argumento de venda:* "Cada dia de atraso em obra tem custo real — aluguel, hospedagem, stress. O Exclusive protege o seu cronograma, não apenas a obra."

---

**Arg. 5 — Sinal de mercado e proteção de reputação mútua (estratégico)**

Toda construtora do mercado faz "qualquer projeto de qualquer tamanho". A D'LIMA é a única que declara publicamente que projetos >200m² têm classificação obrigatória. Isso afasta clientes que buscam menor preço em obra complexa — os mais problemáticos e os que mais danificam reputação quando a entrega falha.

*Argumento de venda:* "Qualquer empresa aceita qualquer obra. Nós estabelecemos o padrão mínimo que garante que o seu projeto será entregue da forma que merece."

> **Insight Pratfall Effect (Aronson):** Admitir uma "limitação" (não aceito obra grande em protocolo inferior) aumenta credibilidade e liking. Somado à *Scarcity* de Cialdini (vagas limitadas por capacidade operacional real), cria frame de autoridade difícil de replicar por concorrentes que aceitam tudo.

---

**Resumo executivo das linhas:** O modelo está correto na estrutura. O que diferencia não é a tabela de linhas — é a Trava de Segurança comunicada como expertise, não como burocracia. Documente os processos internos de cada linha primeiro. Lance os pitches com linguagem de proteção técnica, não de produto premium.

Você tem benefícios, não tem história. O cérebro humano não compra benefício — compra transformação vivida por alguém parecido com ele. Falta uma história real de UMA família (nome, foto, antes/depois). A sequência emocional correta é: **Sonho → Bloqueio → Herói que guia → Prova concreta → Ação**. O Leonan não é o herói da história — a família é o herói. O Leonan é o guia.

**Thiago Finch — Funis, Tráfego Pago, Conversão**
Sem VSL, a conversão sangra. Para decisão de alto valor (casa = maior compra da vida), vídeo de 2-3 min do Leonan acima do formulário aumenta conversão em 60-80%. Também falta qualificação progressiva antes do formulário e audiência de remarketing de quem chegou na seção de faixas mas não converteu.

**Dan Kennedy — Direct Response**
Se não tem prova social específica e mensurável, não tem venda em ticket alto. Quantas famílias atendidas? Qual valor médio do subsídio? Qual prazo médio? Especificidade = credibilidade. E urgência = zero. "Atendemos no máximo 4 famílias por mês" — se for verdade, coloca.

**Alex Hormozi — Ofertas Irresistíveis**
A oferta está invisível. Construção com valor empilhado: simulação grátis + acompanhamento em tempo real + ART assinada + garantia de prazo + projeto personalizado. Quando você empilha o valor e coloca garantia, o preço para de ser objeção.

**Gary Halbert — Emoção Bruta**
Mostre, não descreva. A foto da entrega das chaves está sozinha sem contexto. Embaixo dela: *"Essa é a Ana e o Carlos. Pagavam R$780 de aluguel há 7 anos. Hoje pagam R$390 de parcela — e a casa é deles."* E no CTA final, coloca o cálculo real: *"Se você paga R$800/mês → em 5 anos você entregou R$48.000 ao dono do imóvel."*

**Russell Brunson — Value Ladder**
Os 4 passos estão vendendo processo quando deveriam vender facilidade. Reframe: Simulação → *"Você descobre em 24h se tem direito ao subsídio"*. Carta de crédito → *"A Caixa aprova. Você não precisa entender de burocracia."* Projeto + Obra → *"Você acompanha tudo pelo celular. Sem surpresa."* Entrega → *"O dia que muda a história da sua família."*

**Ícaro de Carvalho — Posicionamento, Autoridade**
O diferencial real (engenheiro local + sistema de acompanhamento + especialista MCMV no norte do ES) está enterrado. O sistema de acompanhamento em tempo real precisa de screenshot real com legenda. É um diferencial de nível nacional que está sendo desperdiçado.

**Mercado Imobiliário (Tom Ferry / Gary Keller)**
Lead qualificado para contrato precisa de 3 coisas antes de assinar: confiança, prova e risco zero percebido. Implementar: mapa visual de área de atendimento, calculadora interativa de subsídio (Results in Advance de Frank Kern), e timeline visual do processo com prazos reais.
